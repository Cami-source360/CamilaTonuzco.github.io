Project 03

IoT Greenhouse System

Sensor-based plant monitoring and irrigation system with MQTT and cloud integration

Presented at the 7th IEEE Colombian Conference on Automatic Control (CCAC 2025) as part of the projects awarded in the CO3 National Contest.

Project Details

Completed: June 2025

System Overview

This project aimed to design and build a solar-powered smart greenhouse system capable of remotely monitoring and irrigating two plants. Each plant features three sensors (temperature, humidity, and luminosity), a dedicated LED for light control, and a solenoid valve for independent irrigation control. A single water pump supplies the system, with flow routing managed through the valves.

Mechanical and Electrical Design

The greenhouse was built using laser-cut flat sheets combined with 3D-printed components to achieve a balance between structural rigidity and modular design.

The electronic system was initially designed in KiCad, enabling PWM control for LEDs and MOSFET/relay switching for valves and the pump. A custom PCB was planned, but a universal PCB was used in the final prototype due to time constraints. The system migrated from STM32 to ESP32 to accelerate development and improve wireless integration.

Embedded System

The ESP32 was programmed using FreeRTOS, enabling concurrent execution of three main tasks: actuator control, sensor acquisition, and MQTT communication.

Each sensor reading and actuator state was timestamped using an onboard RTC and published to a local MQTT broker. Event synchronization messages were also received and parsed to coordinate system behavior.

Backend Architecture

The system architecture is composed of two backend layers.

Node.js Backend: Acts as a bridge between the ESP32 and the database. It subscribes to MQTT topics and logs incoming state variables into a Supabase PostgreSQL database. It also monitors event variables and republishes them for the ESP32.

FastAPI Backend: Handles POST requests from the front-end (Grafana). All user-triggered updates are routed through this service to ensure synchronization and data consistency.

Front-End and Visualization

Grafana was used to visualize time-series data directly from Supabase. All user interface updates are routed through the FastAPI backend instead of direct database writes to ensure synchronization between user commands and device state.

State and Event Variable Architecture

A key concept of the system is the separation between state variables and event variables.

State variables (e.g., valve status) are updated exclusively by the ESP32, while event variables are set by the user (e.g., irrigation request). When an event variable changes, it is published by the Node.js backend, allowing the ESP32 to react and synchronize physical behavior accordingly.
